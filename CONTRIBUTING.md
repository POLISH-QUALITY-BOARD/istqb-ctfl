# Współtworzenie projektu ISTQB CTFL

Dziękujemy za zainteresowanie rozwojem tego repozytorium.

Ten projekt zawiera zlokalizowane na język polski materiały edukacyjne związane z ISTQB Certified Tester Foundation Level (CTFL). Mile widziane są wkłady poprawiające jakość, czytelność, spójność i użyteczność materiałów.

## Jak możesz pomóc

Zapraszamy do wsparcia polegającego na:

- poprawa tłumaczeń i ujednolicenie terminologii,
- poprawa literówek, błędów językowych i formatowania,
- uzupełnianie brakujących tematów,
- porządkowanie struktury treści,
- wskazywanie nieścisłości, nieaktualnych informacji lub niespójności,
- dodawanie pomocnych tabel, zestawień lub diagramów, jeśli zwiększają wartość edukacyjną.

## Zanim zaczniesz

Przed otwarciem pull requesta:

1. Sprawdź, czy temat nie został już opisany w repozytorium.
2. Dla większych zmian otwórz najpierw issue.
3. Dbaj o to, aby zmiany były możliwe do szybkiego sprawdzenia.
4. Upewnij się, że Twoje propozycje pasują do edukacyjnego charakteru projektu.

## Zasady dotyczące treści

Ponieważ repozytorium ma charakter edukacyjny, prosimy o:

- jasny i zwięzły język,
- poprawność merytoryczną,
- spójność z terminologią ISTQB,
- neutralny ton,
- własne sformułowania zamiast kopiowania treści sylabusów lub materiałów chronionych prawem autorskim.

Przy pracy nad terminologią korzystaj z oficjalnego słownika ISTQB w języku polskim:
https://glossary.istqb.org/pl_PL/search?term=&attributes=%5B%22name%22%2C%22abbreviation%22%2C%22synonyms%22%5D&exact_matches_first=true&second_language=en_US

Jeżeli dany termin występuje w słowniku, prosimy o stosowanie jego oficjalnej wersji lub możliwie najbliższego odpowiednika zgodnego ze słownikiem.

## Struktura repozytorium

Prosimy o zachowanie istniejącej struktury plików i nazewnictwa.

Jeśli dodajesz nowe materiały, umieszczaj je w pobliżu powiązanych treści i utrzymuj spójność z aktualnym układem repozytorium.

## Proces wnoszenia zmian

1. Utwórz nową gałąź (branch) dla swojej zmiany.
2. Wprowadź poprawki.
3. Sprawdź formatowanie, linki (renderowanie Markdown jeszcze nie działa)
4. Otwórz pull request z jasnym opisem zmian.

Proponowane nazwy gałęzi:

- `fix/opis-zmiany`
- `feat/opis-zmiany`
- `docs/opis-zmiany`

## Wymagania dla pull requestów

Dobry pull request powinien:

- jasno opisywać cel zmiany,
- zawierać link do issue, jeśli istnieje,
- mieć możliwie mały zakres,
- nie łączyć niezwiązanych ze sobą poprawek,
- zawierać zrzuty ekranu tylko wtedy, gdy zmiana dotyczy wyglądu lub prezentacji treści.

## Zgłaszanie problemów

Jeśli zauważysz błąd, ale nie chcesz od razu go poprawiać, otwórz issue i podaj:

- co jest nie tak,
- gdzie problem występuje,
- czego oczekiwałeś,
- dodatkowy kontekst lub zrzut ekranu, jeśli pomaga w zrozumieniu problemu.

## Proces przeglądu

Opiekunowie projektu mogą zasugerować zmiany przed scaleniem. Prosimy o otwartość na uwagi i gotowość do aktualizacji swojej gałęzi.

## Kodeks współpracy

Prosimy o uprzejmość, konstruktywność i wzajemny szacunek w dyskusjach. Nie akceptujemy hejtu, spamowania ani zachowań ofensywnych.

## Licencja

Wnosząc wkład do projektu, akceptujesz fakt, że Twoje zmiany mogą być rozpowszechniane na warunkach obowiązującej licencji repozytorium.
